# API Integration Setup Guide

## 🎯 Overview

The AI Football Research System now supports:

- ✅ **OpenAI LLM** - Intelligent analysis synthesis with GPT-4
- ✅ **API-Football** - Live match data (optional, falls back to mock data)

Both integrations have **automatic fallbacks** so the system works without API keys.

---

## 🔑 Step 1: Add Your API Keys

### Option 1: Manual Edit (Recommended)

1. Open `backend/.env` in your editor
2. Add your API keys:

```env
OPENAI_API_KEY=sk-proj-xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
FOOTBALL_API_KEY=your_api_football_key_here
ENVIRONMENT=development
LOG_LEVEL=INFO
```

### Option 2: PowerShell Command

```powershell
cd backend

# Add OpenAI key
(Get-Content .env) -replace 'OPENAI_API_KEY=', 'OPENAI_API_KEY=sk-proj-YOUR_KEY_HERE' | Set-Content .env

# Add API-Football key
(Get-Content .env) -replace 'FOOTBALL_API_KEY=', 'FOOTBALL_API_KEY=YOUR_KEY_HERE' | Set-Content .env
```

---

## 🔐 Getting API Keys

### OpenAI API Key (Required for LLM)

1. Go to: https://platform.openai.com/api-keys
2. Sign in or create account
3. Click "Create new secret key"
4. Copy the key (starts with `sk-proj-...`)
5. Add to `.env` file

**Cost**: Pay-as-you-go (GPT-4: ~$0.03/request)

### API-Football Key (Optional)

1. Go to: https://www.api-football.com/
2. Sign up for free account
3. Get your API key from dashboard
4. Add to `.env` file

**Free Tier**: 100 requests/day

## 🔄 How Fallbacks Work

### Without Any API Keys

```
User Query → Agent (rule-based logic) → Mock Data → Response
```

### With OpenAI Only

```
User Query → Agent (GPT-4 analysis) → Mock Data → Response
```

### With Both Keys

```
User Query → Agent (GPT-4 analysis) → Live API Data → Response
```

## 💡 Cost Optimization

### OpenAI Usage

- Each analysis = 1 API call to GPT-4
- Estimated cost: $0.02-0.03 per analysis
- Use GPT-3.5-turbo for cheaper alternative (change in `football_agent.py`)

### API-Football Usage

- Each query = 6 API calls (form × 2, h2h, league × 2, stats × 2)
- Free tier allows ~16 analyses per day
- Consider caching for production

## 🐛 Troubleshooting

### "OPENAI_API_KEY not set"

- Check `.env` file exists in `backend/` folder
- Ensure key starts with `sk-proj-` or `sk-`
- Restart backend server

### "HTTP 401: Unauthorized" (API-Football)

- Verify API key is correct
- Check free tier limits (100/day)
- Try test endpoint: `https://v3.football.api-football.com/status`

### "Module 'langchain_openai' not found"

```powershell
cd backend
.\.venv\Scripts\Activate.ps1
pip install langchain-openai
```
